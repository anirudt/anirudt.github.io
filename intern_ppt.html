My work

  • Worked on an off-target test framework.
  • Visualising plausible scenarios and implementing the test case scenarios and sufficient validation as well.
  • Every new feature addition/bug fix needs to satisfy all the off-target test cases.
  • Worked on Qualcomm's proprietary version of Radio Interface Layer [RIL].

Test Framework

  • Entire framework software is written in C/C++.
  • Framework consisting of several modules, to test several components present in the Linux Data Architecture.

  • Each suite consists of a Makefile to compile the files with the requisite header files, and one Makefile to link them all.
  • Consisting of a simulator acting as a modem, to relay dummy responses back to RIL.

Enhancements

  • Added multiple test cases to test Qualcomm's RIL in terms of Dual IP and Single IP calls, with the concept of Multi-PDN. Also, granular validation was added in terms of unsolicited events.
  • Made the simulator side support a new feature called "Partial-Retry" and ensured test cases for the same. Fixed a bug in the proprietary RIL code to accommodate correct passing of IP-family through APIs.

Enhancements

  • Added the concept of an attach PDN list in the off target simulator. This is used for modem's initial attach. A lot of cases regarding skipping reattach, success and failure cases were drafted.
  • Several querying APIs were made to query the attach PDN list from the test cases.

Enhancements

  • Added simulator support for DSD Registrations, and provided foolproof responses instead of providing dummy responses.
  • Randomized the doling out of IPs in case of Multi-PDN calls, to prevent clashes between IPs.

Some Tweaks

  • All the output is piped to a master log file. Most developers would work only on a single test case at a time.
  • Hence, developed a Perl script which would parse the log file and pipe out a "test_case_file.log" file for convenience.
