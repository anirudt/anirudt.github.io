Vim Ideology

Date: 19th October, 2014

There are basically 3 modes functioning within Vim. One, a middle mode used for aligning the basic format of the entire document, Two, the main mode for editing and adding stuff, and Three, a mode for exiting, saving, etc.

  • From 1 to 2: Press i.
  • From 1 to 3: Press ‘:’ and type commands.
  • From 2 to 1: Press escape once, or as many times.
  • Press ‘>>’ for aligning. Keep a number ‘n >>’ for shifting the next n lines of code by the same.
  • ‘Ctrl+A’ for going to the last letter of the line in Middle Mode[1].
  • ‘dd’ for deleting the present line.
  • ‘.’ for doing what you just did again.
  • h,j,k,l are arrow keys in the middle mode.

List of Unix Commands used in Bandit/CTFs

Date: 18th October, 2014

A list of Linux tools used for Bandit. Some specific usages are also provided.

  • du - used for estimating file space usage by a file
  • file ./* – used for giving the file types of all the files present in the present working directory
  • ls, cd, cat, find - pretty obvious/frequent to use
  • Spaces in the file-name can be overridden by use of a backslash.
  • grep – match lines corresponding to a particular pattern. Very useful command, with lots of flags enabled as well.
  • base64 – base64 encode/decode data. Use the -d flag for decoding.
  • sort – sorts lines of text files
  • uniq – output/omit repeated files. Can be modelled well using a number of flags. See our Frequency Analysis post for the same.
  • tr – translate characters. Helps to map a string/or encode it into a particular cipher. Used for cryptographical purposes.
  • strings - Print the strings of printable characters in the file
  • gzip, bzip2 – used to compress/expand files. Note the file type before doing anything.
  • xxd – Gives the memory hex dump.
  • cp – Copies a file from a source to the destination
  • mv – Moves a file from a source to a destination. Used especially for renaming files.
  • nc – Makes arbitrary TCP/UDP connections possible and listens. Has a plethora of functions/flags, utilities. Search for netcat.
  • openssl – OpenSSL Command Line Tool. Basically, s_client is really useful.
  • s_client – Used for connecting to a particular host especially through a particular port. Used with quite a few flags.
  • nmap – Used to know the status of all ports in the machine. Very useful for determining functions of the ports and to protect subsequent unprotected ports from attacks.
  • Setuids are binary executables. Search on Wikipedia for more information about their user access, etc. They can be executed by ./setuid_name {arguments_if_any}. Also, {strings} can be used to find the working inside.
  • cron, crontab - pretty self explanatory. The time machine of the system.

Smash the stack

Date: 22nd September, 2014

One of the main concepts involved in Capture The Flag stage, named Cusco, typically revolved around smashing the stack. The earlier stages involved finding clues about the password within the live memory. This one prevented any such processes, and allowed a longer password than the earlier one. Let us talk about what the term exactly means, so that we can effectively use it for future purposes.
What does it mean?
Smashing the stack, involves storing input into a stack, which will overflow the existing number of bytes in the stack. This thus, will creep into the memory locations of the neighboring registers, executing which, we can manipulate the program counter. Another hack procedure, could be to search for a return command, as the program counter will point to where the stack pointer is, after execution. The strcpy function also has bugs, which can be exploited.

Stack pointer and memory storage conventions

Date: 20th September, 2014

A noteworthy point of discussion is the difference in conventions of the stack pointer direction. Following are the points to note:

  • The stack pointer moves about, throughout the stack, and is the head of the stack. Any pushing/popping happens where it points to.
  • Memory allocation is from bottom(least) to top(most).
  • When the caller function meets the calling statement of another function, the stack pointer goes a few required bytes down, and begins execution of the statements. Then, it will come back and follow all steps again.
  • The register values stored inside the functions also depend on the lexical scope that they’re based at. Hence, if any value needs to be stored, decrement/increment the sp accordingly and store the values there, to be popped at the time of reference.

Jumping into CTFs

Date: 18th September, 2014

My first play and learn session with CTF was really exhilarating. This is a wonderful way to learn Assembly Language, and to gain a good understanding of what ‘Capture the Flag’ actually is. Following is a synopsis of the tutorial concerned with it, which forms the primary building block of what comes after this.
Some Assembly commands were:

  • tst – testing whether a number is non zero or not
  • mov – moving the first operand into the second operand [more appropriately, the value]
  • add – storing the sum of the two operands into the second one [will take only the 4 bits and ignore any overflow bits]
  • inc/dec – inc(dec)rementing the value of the register
  • incd/decd – inc(dec)rementing the value of the register by 2 bytes
  • jnz – jumping if the register dealt with in the previous line of code is not zero
  • jeq/jz – jumping if the register dealt with in the previous line of code is zero, or equal( essentially, both mean the same, as the cmp function hands over a ‘zero flag’ once the two numbers are equal.)
  • cmp – compares two values and stores the result in a status flag
  • ret – returns the control to the caller function
  • swpb – swaps the bits [in a nibble]
  • bis - Binary subtracts and stores the lesser significant 4 bits/ Also called Bit set, or the ‘OR’ operation.
  • sxt – Extends the word to binary form, to enable mathematical operations on the register, like addition, multiplication, etc.

Some basic conventions are:
  • #x – refers to a constant value to be added, operated or compare to.
  • @R – works like the *ptr form, which accesses the value stored within a register pointed to by R.

Basic Learnings and things to carry forward
  • sr – this is the Status Register, hovering the mouse over it would indicate c-> carry flag is set, else it means that the zero flag is not set yet
  • The Disassembly contains all the code available within the kernel for manipulation and reading
  • The Memory Area will contain the values stored inside the different registers in their specific addresses
  • The Instruction Area exactly contains which instruction is going to be executed. This can be accomplished properly with step(s), break(b) etc.
  • The console can be reached for help by using the ‘help’ command
  • In the tutorial, r12 – stores the number of letters the entered password contains
  • All details are stored in hex, base 16

Key Idea Find the code line which needs to be reached, and try to backtrack your events from there for better understanding of what needs to be achieved. Analyse the function properly.
