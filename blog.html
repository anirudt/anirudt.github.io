Jumping into CTFS

Date: 18th September, 2014

My first play and learn session with CTF was really exhilarating. This is a wonderful way to learn Assembly Language, and to gain a good understanding of what ‘Capture the Flag’ actually is. Following is a synopsis of the tutorial concerned with it, which forms the primary building block of what comes after this.
-Assembly Functions

  • tst – testing whether a number is non zero or not
  • mov – moving the first operand into the second operand [more appropriately, the value]
  • add – storing the sum of the two operands into the second one [will take only the 4 bits and ignore any overflow bits]
  • inc/dec – inc(dec)rementing the value of the register
  • incd/decd – inc(dec)rementing the value of the register by 2 bytes
  • jnz – jumping if the register dealt with in the previous line of code is not zero
  • jeq/jz – jumping if the register dealt with in the previous line of code is zero, or equal( essentially, both mean the same, as the cmp function hands over a ‘zero flag’ once the two numbers are equal.)
  • cmp – compares two values and stores the result in a status flag
  • ret – returns the control to the caller function
  • swpb – swaps the bits [in a nibble]
  • bis - Binary subtracts and stores the lesser significant 4 bits/ Also called Bit set, or the ‘OR’ operation.
  • sxt – Extends the word to binary form, to enable mathematical operations on the register, like addition, multiplication, etc.
